# Totoro-paradise 使用说明

## 📖 项目简介

Totoro-paradise 是一个改进版的龙猫学校跑步打卡工具,支持自定义跑步时长(精确到秒),无需使用 MITM(中间人攻击)。

### ✨ 改进功能

- ✨ **新增自定义跑步时长功能** - 可精确到秒
- ✨ **支持分钟和秒数分别输入** - 更灵活的时间控制
- 🐛 **修复自定义时间导致的卡死问题**
- 📚 **添加启动脚本** - `start.sh` 和 `launch-app.sh`
- 📚 **添加完整的使用说明文档**

## 📝 使用指南

### 1. 登录系统

首次使用需要登录龙猫学校账号:

1. 访问 `http://localhost:3000`
2. 输入学校、学号、密码等信息
3. 点击登录

### 2. 选择跑步路线

登录后进入路线选择页面:

- **下拉选择**: 从下拉菜单中选择预设路线
- **随机路线**: 点击"随机路线"按钮自动选择
- **地图查看**: 地图显示路线的大致位置(仅供参考)

### 3. 设置跑步时长

#### 使用默认时间

不勾选"自定义跑步时间",使用系统默认的时间范围。

#### 自定义跑步时长

1. 勾选"自定义跑步时间"开关
2. 在"分钟"输入框中输入分钟数 (例如: 15)
3. 在"秒"输入框中输入秒数 (例如: 30)
4. 总时长 = 15分30秒

**注意事项:**
- 分钟和秒数都必须 ≥ 0
- 建议设置合理的跑步时间(5-60分钟)
- 自定义时间精确到秒,可以更灵活地控制

### 4. 开始跑步

1. 点击"开始跑步"按钮
2. 确认跑步路线和时长信息
3. 点击"确认开跑"

跑步过程中:
- 会显示进度条
- 可以看到已用时间和总时间
- 请勿在跑步过程中刷新页面或关闭浏览器

### 5. 跑步完成

跑步完成后:
- 页面会提示"跑步完成"
- 可以在龙猫学校 App 中查看跑步记录

## 🔧 常见问题

### Q: 启动后无法访问页面?

A: 检查以下几点:
1. 确认服务已启动: `docker-compose ps`
2. 检查端口是否被占用: `lsof -i :3000`
3. 查看日志: `docker-compose logs -f`

### Q: 地图无法显示?

A: 可能是高德地图 API 配额已用完,不影响跑步功能。

### Q: 跑步时卡死?

A: 已修复自定义时间的卡死问题,确保使用最新版本。

### Q: 如何修改默认时间范围?

A: 可以在跑步页面的自定义时间输入框中设置所需时长。

### Q: 如何取消正在进行的跑步?

A: 目前不支持取消,请等待跑步完成。
